# ブログサイト要件定義書

## 1. プロジェクト概要

### 1.1 目的

個人用の技術ブログサイトを構築する。プログラミングや技術的な内容を発信するためのプラットフォーム。

### 1.2 技術スタック

| 項目           | 技術                 |
| -------------- | -------------------- |
| フレームワーク | Next.js (App Router) |
| スタイリング   | Tailwind CSS         |
| コンテンツ管理 | Markdownファイル     |
| ホスティング   | Vercel               |
| UIライブラリ   | shadcn/ui            |

---

## 2. ページ構成

| ページ   | パス            | 説明                           |
| -------- | --------------- | ------------------------------ |
| ホーム   | `/`             | 記事一覧を表示するトップページ |
| 記事詳細 | `/posts/[slug]` | 個別記事の表示ページ           |

---

## 3. 機能要件

### 3.1 コンテンツ管理

- Markdownファイルによる記事管理
- フロントマター（YAML）でメタデータを管理
  - title: 記事タイトル
  - date: 公開日
  - tags: タグ（配列）
  - description: 記事の説明

### 3.2 記事表示機能

- Markdownのレンダリング
- シンタックスハイライト（コードブロックの色分け表示）

### 3.3 検索機能

- 記事タイトル・本文からのキーワード検索
- クライアントサイドでの検索実装

### 3.4 ダークモード

- ライト/ダークモードの切り替え機能
- システム設定に応じた初期表示
- ユーザーの選択を保持

---

## 4. デザイン要件

### 4.1 デザイン方針

- ミニマル・シンプル
- 余白を活かしたクリーンなレイアウト
- 読みやすさを重視したタイポグラフィ

### 4.2 レスポンシブ対応

- モバイル、タブレット、デスクトップに対応
- モバイルファースト設計

### 4.3 カラースキーム

- ライトモード：白基調
- ダークモード：暗色基調

### 4.4 UIライブラリ

- ボタンやカードなど、UIパーツはshadcn/uiを利用

---

## 5. SEO対応

### 5.1 対応範囲（最小限）

- 基本的なメタタグ（title, description）
- ページごとのメタデータ設定

---

## 6. 非機能要件

### 6.1 パフォーマンス

- 静的生成（SSG）による高速表示
- 画像の最適化

### 6.2 アクセシビリティ

- セマンティックなHTML構造
- キーボード操作対応

---

## 7. ディレクトリ構成（案）

```
/
├── app/
│   ├── layout.tsx        # ルートレイアウト
│   ├── page.tsx          # ホームページ
│   └── posts/
│       └── [slug]/
│           └── page.tsx  # 記事詳細ページ
├── components/           # 共通コンポーネント
├── content/
│   └── posts/            # Markdown記事
├── lib/                  # ユーティリティ関数
├── docs/                 # ドキュメント
└── public/               # 静的ファイル
```
