# 開発フェーズ

## 概要

本プロジェクトを5つのフェーズに分けて開発を進める。

---

## フェーズ1: プロジェクト基盤構築

### 目標
Next.js + Tailwind CSS + shadcn/uiの環境構築と基本レイアウトの作成

### タスク
- [ ] Next.js (App Router) プロジェクトのセットアップ
- [ ] Tailwind CSSの設定
- [ ] shadcn/uiの初期化・設定
- [ ] 必要なshadcn/uiコンポーネントの追加（Button, Card等）
- [ ] ディレクトリ構成の整備
- [ ] 共通レイアウト（Header, Footer）の作成
- [ ] 基本的なスタイル設定（フォント、カラー変数）

### 成果物
- 動作するNext.jsプロジェクト
- shadcn/ui導入済みの環境
- 共通レイアウトコンポーネント

---

## フェーズ2: Markdown記事システム

### 目標
Markdownファイルから記事を読み込み、表示できる仕組みの構築

### タスク
- [ ] Markdown処理ライブラリの導入（gray-matter, remark等）
- [ ] 記事データ取得関数の実装
- [ ] フロントマター（メタデータ）のパース処理
- [ ] シンタックスハイライトの実装
- [ ] サンプル記事の作成

### 成果物
- 記事取得用のユーティリティ関数
- Markdownレンダリング機能
- テスト用サンプル記事

---

## フェーズ3: ページ実装

### 目標
ホームページと記事詳細ページの実装

### タスク
- [ ] ホームページ（記事一覧）の実装
  - shadcn/ui Cardを使った記事カードコンポーネント
  - 日付順ソート
- [ ] 記事詳細ページの実装
  - 動的ルーティング `/posts/[slug]`
  - 記事本文の表示
  - メタ情報（タイトル、日付、タグ）の表示
- [ ] 静的生成（generateStaticParams）の設定

### 成果物
- 完成したホームページ
- 完成した記事詳細ページ

---

## フェーズ4: 追加機能実装

### 目標
検索機能とダークモードの実装

### タスク
- [ ] ダークモードの実装
  - shadcn/uiのテーマ切り替え機能を活用
  - システム設定の検出
  - LocalStorageによる設定保持
- [ ] 検索機能の実装
  - shadcn/ui Inputを使った検索UI
  - クライアントサイド検索ロジック
  - 検索結果の表示

### 成果物
- ダークモード切り替え機能
- 記事検索機能

---

## フェーズ5: 仕上げ・デプロイ

### 目標
SEO対応、最終調整、本番デプロイ

### タスク
- [ ] SEO対応
  - メタタグの設定（Next.js Metadata API）
  - 各ページのtitle, description設定
- [ ] レスポンシブデザインの最終調整
- [ ] パフォーマンス確認・最適化
- [ ] Vercelへのデプロイ
- [ ] 動作確認・バグ修正

### 成果物
- 本番環境で動作するブログサイト
